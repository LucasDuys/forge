Intelligence

Stop Hook State Machine

Every time Claude tries to exit, the hook reads state and decides: what should happen next?

>_

EXECUTE

Task not yet implemented. Feed implementation prompt with spec requirements and acceptance criteria.

✓

TEST

Code written but not tested. Feed "run tests" prompt. On failure, route to FIX.

🔍

REVIEW

Tests pass but not reviewed. Feed review prompt. On issues, route to FIX.

🔧

FIX

Issues found (test failures or review). Feed targeted fix prompt. Track attempt count.

🐛

DEBUG

Same failure 3x in a row. Switch to systematic 4-phase debugging. Escalate after 3 debug attempts.

⏭

ADVANCE

Task complete. Commit, update frontier, check budget. Route to next task or phase verification.

Self-Improvement

Backpropagation

Trace runtime bugs back to the spec gap that missed them. The system learns from its mistakes.

1

Bug Discovered

Runtime error, failing test, or user report. Run /forge backprop "description"

2

Trace to Spec

Identify which requirement and acceptance criterion missed this case. Pinpoint the spec gap.

3

Update Spec

Add or modify acceptance criteria. Fix the root cause in the specification, not just the code.

4

Generate Regression Test

Auto-create a test that would have caught this bug. Locks in the fix permanently.

5

Verify from Spec

Optionally re-run execution to confirm the fix emerges from updated specs alone. System is now smarter.

Safety

Circuit Breakers

Prevent infinite loops and token waste. The system knows when to stop and ask for help.

Same test fails 3x in a row
Switch to DEBUG mode — 4-phase systematic investigation instead of blind retries
DEBUG mode > 3 attempts
Pause and ask human — save state, explain what was tried, request guidance
Review loop > 3 iterations
Accept with warnings — log unresolved issues, move to next task
No progress for 2 iterations
Stall detection — save state, flag blocker, pause for human input
Token budget exhausted
Graceful exit — save all state, report progress, list remaining tasks
Single task > 20% of budget
Force-complete — commit what exists, flag for human review, move on
